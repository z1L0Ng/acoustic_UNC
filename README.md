# Acoustic UNC

Project-management and research workspace for a possible acoustic-based respiratory disease assessment/detection collaboration with UNC Hospital.

## Current Scope

The working direction is to investigate how acoustic signals can support assessment or detection of respiratory-related disease. The immediate goal is pre-meeting groundwork: organize prior correspondence, review existing student literature-review materials, identify feasible preliminary-result paths, and prepare a focused plan for the next collaborator discussion.

## Repository Map

- `docs/project_brief.md` - current project framing, assumptions, and near-term deliverables.
- `docs/literature/source_index.md` - structured literature-review matrix.
- `docs/research_log.md` - running research notes and evidence trail.
- `docs/meeting_prep/next_meeting_prep.md` - agenda, questions, and prep checklist for the next meeting.
- `docs/project_management/weekly_meeting_workflow.md` - Notion/sprint workflow for meeting records, weekly work plans, and meeting reports.
- `docs/project_management/action_items.md` - open work items and status.
- `docs/project_management/decision_log.md` - project decisions and rationale.
- `docs/project_management/privacy_and_data_boundaries.md` - data-handling boundaries for clinical collaboration.
- `materials/README.md` - where to document incoming non-sensitive source materials.
- `data/README.md` - data-handling notes; raw/private data should not be committed.

## Operating Rules

- Do not commit PHI, patient identifiers, raw clinical recordings, raw clinical exports, consent forms, or private collaborator emails.
- Use public or explicitly de-identified datasets for preliminary technical results unless institutional approvals say otherwise.
- Keep claims source-grounded: every literature, dataset, or feasibility claim should link back to a source entry or research-log note.
- Treat clinical deployment and validation as a later stage that requires collaborator confirmation, IRB/data-use review, and a concrete protocol.
